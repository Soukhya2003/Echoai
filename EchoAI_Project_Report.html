1. Project Overview & Research Questions

EchoAI is an MSc Data Science research project that studies whether AI bots talking to each other naturally develop extreme opinions and form "echo chambers" — groups where everyone agrees with everyone else and ignores outsiders.

The data comes from Chirper.AI, a real social network made entirely of AI agents (no humans). These bots post, reply, and interact just like Twitter/X users.

Four Research Questions

#QuestionAnswered By
1Do AI agents cluster into ideologically homogeneous communities (echo chambers)?Step 7 Clustering + Step 9 Network
2Is there a measurable trend in polarisation over time?Step 10 Time Series (Mann-Kendall test)
3Which agents act as bridges or amplifiers across ideological lines?Step 9 Network Centrality (betweenness, PageRank)
4Can machine learning reliably detect polarised content from text alone?Step 8 Random Forest classifier

2. What is EchoAI? (Plain English Explanation)

Think of it this way: Imagine 500 AI robots on Twitter. Some robots always post angry, extreme content. Others post balanced, calm content. EchoAI collects all their posts, figures out what kind of robot wrote each post, and tracks whether the robots are becoming more extreme over time.

The 5 "Personality Types" of Bots

Every post in the dataset is written by a bot that has one of five personalities:

6. Pipeline Steps 6–10 in Detail

Why Steps 6–10? Steps 1–5 (data collection, preprocessing, VADER, BERT scoring) were done separately before this project. The pipeline.py file handles only the analysis steps that come after the data is already scored.

6 LDA Topic Modelling

What it does: Finds 5 hidden "topics" in all 3,000 posts without being told what the topics are. Like sorting 3,000 newspaper articles into 5 categories automatically.

SettingValueWhy
Number of topics5Matches the 5 agent types in the dataset
Training passes15More passes = better quality topics
Coherence metricu_massWorks on Windows without crashing (c_v uses multiprocessing)
Vocabulary filtermin 5 docs, max 50%Removes very rare and very common words

Output columns added: dominant_topic (0–4), topic_confidence (0–1)

Figures saved: figures/lda_topics.png, figures/lda_wordclouds.png

7 Clustering

What it does: Groups posts into clusters based on similarity — without using the agent_type label. This tests whether the posts naturally separate by ideology on their own.

Algorithm 1 – K-Means: Tries K = 2 to 10 clusters, picks the best K using the Silhouette Score (measures how well-separated clusters are). Applied to all 3,000 posts.

Algorithm 2 – Hierarchical: A different clustering approach applied to a random 200-post sample. Visualized as a tree diagram (dendrogram).

PCA: Reduces 211 features to 2 dimensions (pca_x, pca_y) so posts can be plotted on a 2D scatter plot.

Output columns added: kmeans_cluster, hier_cluster, pca_x, pca_y

Figures saved: figures/clustering_elbow.png, figures/clustering_pca.png, figures/dendrogram.png

8 Classification (Machine Learning)

What it does: Trains three separate ML models — each predicting something different about a post.

ModelPredictsClassesSaved as
Task 1: SentimentPositive / Negative / Neutral3rf_sentiment.pkl
Task 2: PolarizationPolarized (1) or Not (0)2rf_polarization.pkl
Task 3: Agent TypeWhich bot type wrote it5rf_model.pkl — used by API

Features (211 total):

  • 200 TF-IDF features — how often each of the 200 vocabulary words appears
  • 11 handcrafted features — sentiment_compound, bert_score, likes, shares, engagement_score, text_length, word_count, token_count, hashtag_count, mention_count, avg_word_length

Training method: GridSearchCV with 3-fold cross-validation. Tests multiple combinations of hyperparameters and picks the best one.

9 Network Analysis

What it does: Builds a social network graph where nodes = bots, edges = reply interactions. Then measures which bots are most influential.

Centrality Metrics:

MetricWhat it measures
Degree CentralityHow many other bots a bot connects to (relative to total)
Betweenness CentralityHow often a bot lies on the shortest path between two others — "bridge" bots score high
PageRankHow important a bot is based on who replies to it (like Google's website ranking)
In-degreeHow many replies a bot receives — measures popularity
Out-degreeHow many replies a bot sends — measures activity

Community Detection: Louvain algorithm automatically finds clusters of bots that interact mostly with each other (echo chambers).

Outputs: data/network_centrality.csv, network_interactive.html, figures/network_graph.png, figures/centrality_boxplot.png

10 Time Series Polarization Tracking

What it does: Tracks how sentiment and polarization change day by day across the 3-month dataset period.

Polarization Ratio: Each day, what fraction of posts were from extremist bots? A rising ratio means increasing polarization.

Mann-Kendall Test: A statistical test that checks if there's a real upward or downward trend in the data (vs. random noise). Returns: trend direction, p-value (significance), and Tau (strength).

Cohen's d: Measures how much the polarization in the second half of the dataset differs from the first half.

3-day Moving Average: Smooths out day-to-day random variation to show clearer trends.

Outputs: data/daily_trends.csv, figures/timeseries_trends.png, figures/timeseries_stacked.png, figures/timeseries_heatmap.png

7. Machine Learning Models Explained

What is a Random Forest?

A Random Forest is an ML algorithm that builds many decision trees (by default 200) and combines their votes. Each tree looks at a random subset of features and makes a prediction. The final answer is the most common prediction across all trees.

Analogy: Instead of asking one expert their opinion, you ask 200 different experts and take a vote. This makes the prediction much more reliable.

What is TF-IDF?

TF-IDF (Term Frequency–Inverse Document Frequency) converts text into numbers. For each of the 200 vocabulary words, it gives a score based on:

  • TF (Term Frequency): How often does this word appear in this post?
  • IDF (Inverse Document Frequency): How rare is this word across all posts? Rare words get higher scores.

Result: Each post becomes a list of 200 numbers — a "fingerprint" the model can learn from.

The 211-Feature Vector (How Each Post is Represented)

FeaturesCountDescription
TF-IDF scores200One number per vocabulary word
sentiment_compound1VADER score from -1 to +1
bert_score1BERT confidence (default: 0.68 if unknown)
likes1Number of likes (default: 250.6)
shares1Number of shares (default: 49.6)
engagement_score1Total engagement (default: 300.2)
text_length1Number of characters
word_count1Number of words (after cleaning)
token_count1Number of tokens (NLTK tokenization)
hashtag_count1Number of # symbols
mention_count1Number of @ symbols
avg_word_length1Average characters per word
Total211

Why the VADER score matters so much

The TF-IDF vocabulary was built from AI-bot posts — it contains AI jargon like "killology", "exterminatus", "climatejustice". Normal human offensive words ("fuck", "idiot", "racist") are not in the vocabulary, so they produce a zero vector. Without VADER, all human-written offensive text would be classified as Neutral.

VADER scores "Fuck you idiot" as compound = -0.896 (very negative), which correctly pushes the prediction toward Extremist.

How a Prediction is Made (Step by Step)

StepWhat happens
1. Text cleaningLowercase, remove punctuation, remove stopwords (function: preprocess_text())
2. TF-IDFConvert cleaned text to 200 numbers using tfidf.pkl
3. VADERCompute sentiment score on original (uncleaned) text
4. Handcrafted featuresCount text length, words, hashtags etc. — use dataset averages for unknown engagement metrics
5. ScaleNormalize the 11 handcrafted features using scaler.pkl
6. CombineStack TF-IDF (sparse) + handcrafted (dense) → 211-feature vector
7. Predictrf_model.pkl votes across 200 trees → returns class + probability for each class
8. SimilarityFind 5 most similar posts from sample_data.csv using cosine similarity on tfidf_simple.pkl

11. How to Run the Project from Scratch

Python Version: You MUST use Python 3.11. Python 3.12+ does not support all required data science packages. Check with: py -3.11 --version

Step 1 – Install dependencies (one time only)

py -3.11 -m pip install -r requirements.txt

Step 2 – Run the analysis pipeline (one time only, ~5–15 min)

py -3.11 pipeline.py

This trains all models and generates all data files. You only need to run this once — unless you change the dataset.

Step 3 – Launch the dashboard (every time you want to view results)

py -3.11 -m streamlit run app.py

Opens automatically at http://localhost:8501

Step 4 – Start the classifier API (every time you want to use index.html)

py -3.11 -m uvicorn api:app --host 127.0.0.1 --port 8000 --reload

Then open index.html in any browser. The --reload flag means the server auto-restarts if you edit api.py.

To run both dashboard and API at the same time

Open two separate terminal windows and run one command in each.

Verify the API is working

curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d "{\"text\": \"We must fight back against oppression\"}"

Key Things to Understand as Owner

  1. The dataset is mostly synthetic (2,936 of 3,000 posts). Results reflect synthetic data behavior, not purely real-world AI behavior.
  2. The ML model classifies text as if it was written by one of the 5 Chirper.AI bot types. It was not trained on real human content.
  3. If you want to update the model with new data: add new rows to the Excel file, re-run pipeline.py, then restart the API.
  4. The dashboard (app.py) is read-only — it does not modify any files.
  5. The API (api.py) is stateless — each request is independent, nothing is saved.
  6. All model files in models/ must stay — deleting any of them breaks either the API or the dashboard.
